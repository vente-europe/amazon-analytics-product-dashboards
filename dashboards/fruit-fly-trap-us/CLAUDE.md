# Fruit Fly Trap — US (Detailed Dashboard)

## Purpose

Deep-dive market analysis for the Fruit Fly Trap category on Amazon US. 207 ASINs across 4 segments (Lure, Sticky Traps, Passive Attractor, Electric).

## Tech Stack

- Dashboard Hub (`/Console/`) — vanilla HTML + JS + Chart.js 4.4.4
- Tabs 1-2: **auto-computed from X-Ray + Sales CSVs** at runtime (zero hardcoded values)
- Tab 3: **AI-analyzed VOC data** stored in `dashboard.json` (generated by `scripts/build_voc.py`)
- Tab 4: KW Analysis data in `addonTabs`

## Data Pipeline

- **12M calculation:** 80 ASINs from actual sales data (summed daily), 127 ASINs projected using seasonality index derived from those 80
- **Export month:** February (exportMonth: 1) — a trough month for fly traps, seasonality correction is critical
- **Seasonality:** Highly seasonal category — Electric traps peak ~2.3× in summer, Lure ~2× 
- **Revenue:** 12M units × listed ASIN price (from X-Ray)

## Reviews Analysis

- **5,315 unique reviews** merged from 5 scraper files + terro_reviews.json + hs_reviews.json
- **Star distribution:** 1★: 2309, 2★: 1487, 3★: 1257, 4★: 117, 5★: 145 (avg 1.93 — heavily skewed negative due to scraper bias)
- **Theme tagging:** 13 themes via regex (no_effect, effective, diy_better, etc.)
- **Build script:** `scripts/build_voc.py`

## Known Issues

- **Review data skew:** Scraped datasets (dataset_amazon*.json) contain mostly 1-3 star reviews. Only terro_reviews.json and hs_reviews.json have 4-5 star reviews. Avg rating 1.93 is not representative of actual product ratings — it reflects the scraping filter, not market reality.
